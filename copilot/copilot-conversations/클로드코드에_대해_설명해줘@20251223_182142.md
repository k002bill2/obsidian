**ai**: 클로드 코드는 Clean Code 원칙을 따르는 프로그래밍 스타일과 실천 방법을 중심으로 설명할 수 있습니다. 주요 내용은 다음과 같습니다:

1. **명확한 이름 사용**: 변수, 함수, 클래스 등의 이름은 의미를 명확하게 표현해야 합니다. 예를 들어 `calculateTotalPrice` 대신 `totalCost` 보다 명확한 이름을 선택합니다 ([^1]).

2. **작은 함수 만들기**: 각 함수는 한 가지 역할만 수행하도록 설계되어야 하며, 가능한 한 짧고 간결해야 합니다. 이는 코드의 유지보수성을 높이고 재사용성을 증가시킵니다 ([^2]).

3. **자주 변경되는 부분은 분리하기**: 변동성이 높은 부분과 안정적인 부분을 구분하여 관리합니다. 이를 통해 코드의 안정성과 유지보수성을 향상시킬 수 있습니다 ([^3]).

4. **문서화 중요성**: 코드 주석과 문서는 이해도를 높이고, 다른 개발자들이 쉽게 코드를 읽고 이해할 수 있도록 합니다. 중요한 부분은 명확한 설명을 달아야 합니다 ([^4]).

5. **테스트 중심 개발(TDD)**: 테스트 케이스를 먼저 작성하고 그에 맞춰 코드를 작성하는 방식으로, 코드의 정확성을 보장합니다 ([^5]).

**ai**: Claude Code는 Anthropic에서 제공하는 AI 도구로, 개발자들이 프로젝트를 효율적으로 진행할 수 있도록 설계되었습니다. 이 가이드북은 사용자가 Claude Code를 효과적으로 활용하기 위해 필요한 설정과 문서화 방법을 설명합니다.

### 핵심 구성 요소

1. **CLAUSE.md**: 프로젝트의 일반적인 가이드라인을 정의하는 중요한 파일입니다. 여기에는 프로젝트 개요, 디렉토리 구조, 코드 스타일, Git 워크플로 등이 포함됩니다.
   - **예시**:
     ```markdown
     # Project Context for Claude Code

     ## 🎯 Project Overview
     - **Name**: [프로젝트 이름]
     - **Purpose**: [프로젝트 목적]
     - **Tech Stack**: [사용 기술]
     - **Current Phase**: [현재 개발 단계]

     ## 📁 Project Structure
     [디렉토리 구조와 각 폴더의 역할 설명]

     ## 🛠️ Development Guidelines

     ### Code Style
     - Language: [JavaScript/Python/etc]
     - Style Guide: [ESLint/Prettier 설정]
     - Naming Conventions:
       - Functions: camelCase
       - Classes: PascalCase
       - Constants: UPPER_SNAKE_CASE

     ### Git Workflow
     - Branch naming: feature/[feature-name]
     - Commit format: [type]: [description]
       - feat: 새 기능
       - fix: 버그 수정
       - docs: 문서 수정
       - refactor: 코드 리팩토링
     ```

2. **Skills & Agents**: 프로젝트에서 사용할 수 있는 다양한 Skills와 Sub-agents를 정의합니다.
   - **SKILL.md 파일**:
     ```markdown
     # code-reviewer Skill

     ## Purpose
     Perform automated code reviews and provide feedback.

     ## Commands
     - `review [file]`: Review the specified file.
     ```

3. **Level 1: 핵심 설정 문서**: 프로젝트의 기본적인 설정을 정의합니다.
   ```
   your-project/
   ├── CLAUDE.md                    # ⭐ 프로젝트 가이드라인 (가장 중요)
   ├── .mcp.json                    # MCP 서버 연결 설정
   ├── .claudecode.json             # 권한 및 훅 설정
   └── PRD.md                       # 프로젝트 요구사항
   ```

4. **Level 2: Skills & Agents 문서**: 다양한 Skills와 Sub-agents를 구현하고 관리합니다.
   ```
   ├── .claude/
       ├── skills/                  # 🌟 Agent Skills
       │   ├── code-reviewer/
       │   │   └── SKILL.md
       │   ├── test-runner/
       │   │   └── SKILL.md
       │   └── docs-generator/
       │       └── SKILL.md
       ├── agents/                  # 🤖 Sub-agents
       │   ├── frontend-specialist.md
       │   ├── backend-architect.md
       │   └── test-engineer.md
       └── commands/                # 커스텀 명령어
           ├── review.md
           ├── deploy.md
           └── test.md
   ```

### 주요 설정 및 권장 사항

1. **Planning Mode**: 계획 없이 시작하면 프로젝트가 실패할 수 있으므로 필수입니다.
2. **Skills 자동 활성화 Hook 시스템**: Skills를 만들지만 사용하지 않으면 효과적이지 않습니다.
3. **Dev Docs 3-파일 시스템**: Claude가 길을 잃는 것을 방지하기 위해 중요한 문서들을 정리합니다.

**ai**: # 미드저니로 나만의 작업실 구축하기

이 가이드에서는 디스코드 개인 서버를 생성하고, 미드저니 봇(Midjourney Bot)을 초대하여 사용할 수 있는 방법에 대해 자세히 설명합니다. 이를 통해 나만의 작업실을 만들고, 다양한 기능을 활용할 수 있습니다.

## 1. 디스코드 개인 서버 만들기 및 관리

### 서버 생성 단계
1. **서버 추가하기**: 디스코드 왼쪽의 플러스(+) 아이콘을 클릭합니다.
2. **'직접 만들기'**를 선택한 후, **'나와 친구들을 위한 서버'**를 클릭합니다.
3. 원하는 서버 이름(예: 전용 미드저니)을 입력하고 **'만들기'**를 누르면 생성이 완료됩니다.

### 미드저니 봇 초대
1. 미드저니 공식 서버로 이동하여 오른쪽 멤버 목록에서 **'Midjourney Bot'**을 찾아 클릭합니다.
2. '앱 추가' 버튼을 누른 후 **'서버에 추가'**를 선택합니다.
3. 조금 전 만든 개인 서버를 선택하고 '계속하기'와 **'승인'**을 차례로 누르면 개인 서버에서 미드저니를 사용할 준비가 끝납니다.

### 서버 삭제 방법
만약 서버를 지우고 싶다면 서버 아이콘 우클릭 > 서버 설정 > 일반 메뉴에서 가장 아래에 있는 **'서버 삭제'**를 클릭하고 서버 이름을 입력하면 됩니다.

## 2. 미드저니 설정(/settings) 상세 내용

채팅창에 /settings를 입력하면 미드저니의 다양한 기능을 설정할 수 있는 창이 나타납니다.

- **버전 선택**: 현재 기본값인 Model V6를 포함하여 V1부터 V6까지 선택 가능합니다. 최신 버전일수록 결과물의 퀄리티가 훨씬 좋습니다 [^2].
- **스타일라이즈(Stylize) 모드**: 스타일 적용 강도를 Low, Med, High, Very High 4단계로 조절할 수 있으며, 일반적으로는 기본값을 유지합니다 [^2].
- **공개 및 스텔스 모드**: 기본은 공개 모드(Public)이며, 프로 플랜(Pro Plan) 이상 사용자는 결과물을 남들에게 숨길 수 있는 스텔스(Stealth) 모드를 사용할 수 있습니다 [^2].
- **속도 모드**: 플랜에 따라 제공되는 패스트(Fast) 모드 시간 내에서 생성 속도를 조절할 수 있습니다 [^2].
- **리믹스(Remix) 모드**: 기존 이미지의 일부를 수정하거나 새로운 요소를 더할 때 사용하는 유용한 기능입니다 [^2].
- **기타 옵션**: 베리에이션(Variation) 모드를 통해 생성된 이미지를 더 다양하게 변화시킬 수 있으며, 로우(Raw) 모드는 버전 5.1 이후부터 적용 가능한 옵션입니다 [^2].

## 3. 사용자 정보 확인(/info)

자신의 계정 상태가 궁금할 때는 /info 명령어를 사용합니다. 이 명령어를 통해 현재 사용 중인 플랜, 남은 패스트 모드 시간, 지금까지 생성한 총 이미지 개수 등을 한눈에 확인할 수 있습니다 [^2].

**ai**: ### 이지게이트와 오픈게이트 비교

이지게이트와 오픈게이트는 각각의 접근 방식과 특징을 가지고 있으며, 선택은 주로 사용 목적과 요구사항에 따라 결정됩니다.

#### 1. **이지게이트 (EaseGate)**
- **특징**: 
  - **사용자 친화적**: 간단하고 직관적인 인터페이스를 제공하여 사용자가 쉽게 설정할 수 있습니다.
  - **설정 단순화**: 복잡한 설정을 자동으로 처리하여 사용자의 작업 효율성을 높입니다.
- **사용 사례**:
  - 웹 애플리케이션의 배포와 관리를 위한 간단한 도구로 적합합니다.

#### 2. **오픈게이트 (OpenGate)**
- **특징**: 
  - **고도화된 기능**: 다양한 설정 옵션과 확장성을 제공하여 복잡한 요구사항을 처리할 수 있습니다.
  - **커스터마이즈 가능**: 사용자가 직접 구성하고 조정할 수 있는 자유도가 높습니다.
- **사용 사례**:
  - 대규모 시스템이나 고급 설정이 필요한 경우 적합합니다.

#### 비교 요소

1. **설정의 복잡성**
   - **이지게이트**: 간단하고 직관적입니다. 사용자가 쉽게 설정할 수 있습니다.
   - **오픈게이트**: 다양한 설정 옵션과 확장성을 제공하므로, 초기 설정은 더 복잡할 수 있습니다.

2. **사용자 친화성**
   - **이지게이트**: 사용자 친화적이고 간단한 인터페이스를 제공합니다.
   - **오픈게이트**: 고급 기능을 필요로 하는 사용자에게 더 적합하며, 초기 학습 곡선이 높을 수 있습니다.

3. **확장성**
   - **이지게이트**: 기본적인 요구사항만 충족할 때 적합합니다.
   - **오픈게이트**: 복잡한 요구사항과 확장성을 필요로 하는 시스템에 더 적합합니다.

4. **커스터마이즈 가능성**
   - **이지게이트**: 제한적인 커스터마이징 옵션을 제공합니다.
   - **오픈게이트**: 사용자가 직접 구성하고 조정할 수 있는 자유도가 높습니다.
