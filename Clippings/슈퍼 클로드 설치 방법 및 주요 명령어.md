윈도우에서는 NPX를 사용하는 로컬 MCP 서버를 사용하는 경우

cmd /c

래퍼(wrapper)를 반드시 함께 적용해야 합니다. 다음은 수정된 코드입니다.

Fortran

복사

"mcpServers": { "context7": { "type":"http","url":"https://mcp.context7.com/mcp" },"sequential-thinking": { "type":"stdio","command":"cmd","args": \[ "/c","npx","@modelcontextprotocol/server-sequential-thinking" \],"env": {} },"magic": { "type":"stdio","command":"cmd","args": \[ "/c","npx","@21st-dev/magic@latest" \],"env": {} },"playwright": { "type":"std io","command":"cmd","args": \[ "/c","npx","@playwright/mcp@latest" \],"env": {} } },

위 코드를 직접 수정하기 어려운 경우 클로드 코드에게 다음과 같이 명령해보셔도 됩니다.

Fortran

복사

└ \[Warning\] \[playwright\] mcpServers.playwright: Windows requires 'cmd /c' wrapper to execute npx └ \[Warning\] \[sequential \- thinking\] mcpServers.sequential \- thinking: Windows requires 'cmd /c' wrapper to execute npx └ \[Warning\] \[magic\] mcpServers.magic: Windows requires 'cmd / c' wrapper to execute npx

복사

claude mcp add \- s user \- \- transport http context7 https:// mcp.context7.com / mcp claude mcp add \- s user sequential \- thinking npx @modelcontextprotocol / server \- sequential \- thinking claude mcp add \- s user magic npx @ 21 st \- dev / magic@latest claude mcp add \- s user playwright npx @playwright / mcp@latest

/workflow

PRD와 기능 요구 사항을 분석 → 포괄적인 단계별 구현 워크플로 생성

/analyze

코드 품질, 보안, 성능 및 아키텍처에 대한 포괄적인 분석

/build

스마트한 오류 처리를 통한 프로젝트 빌드, 컴파일, 패키징

/implement

지능형 전문가 활성화 통한 기능, 구성 요소 및 기능을 구현

/improve

코드 품질, 성능, 유지관리성을 체계적으로 개선

/troubleshoot

체계적인 디버깅 및 문제 조사

/test

테스트 실행, 적용 범위 보고서를 생성, 테스트 품질을 유지

/document

특정 구성 요소, 기능 또는 특징에 대한 문서 작성

/git

지능형 커밋 메시지와 워크플로 최적화를 통한 Git 작업

/design

시스템 아키텍처, API 설계, 구성 요소 사양을 작성

/explain

교육적인 방식으로 코드, 개념, 기술을 설명

/cleanup

쓸모없는 코드와 사용되지 않는 가져오기를 제거, 파일 구조 정리

load

프로젝트 컨텍스트 로드 및 분석 통한 더 나은 이해를 제공

/estimate

개발 작업에 필요한 시간, 노력, 복잡성을 추산

/spawn

복잡하고 여러 단계로 이루어진 작업과 워크플로 조정

/task

복잡한 다중 세션 개발 작업과 기능을 관리

/index

작업에 적합한 명령을 찾는 데 도움 제공
