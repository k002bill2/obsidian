# PM2를 활용한 n8n 및 클라우드플레어 자동 실행 가이드

제시해 주신 소스는 **n8n 로컬 서버를 구축한 후, 컴퓨터를 켤 때마다 n8n과 클라우드플레어(Cloudflare)가 자동으로 실행되도록 PM2를 활용해 자동화하는 방법**을 상세히 설명하고 있습니다. 주요 내용을 단계별로 정리해 드립니다.

## 1. PM2 도입 배경과 목적

컴퓨터 가상화 시스템(도커 등)을 사용할 수 없는 환경에서 **윈도우 부팅 시 n8n 서버와 외부 연결 도메인(클라우드플레어)을 자동으로 구동**시키기 위해 **PM2**라는 프로그램 관리 도구를 사용합니다,. 이는 매번 수동으로 명령어를 입력해야 하는 번거로움을 없애고, 24시간 안정적인 운영을 가능하게 합니다,.

## 2. 주요 설정 단계 및 명령어

• **PM2 및 자동 실행 패키지 설치:** 먼저 Node.js 환경에서 `npm install pm2 -g` 명령어로 PM2를 설치합니다. 그 후, 윈도우 서비스에 등록하여 자동 부팅 기능을 활성화해 주는 `pm2-windows-startup` 패키지를 추가로 설치하고 설정을 진행합니다,.
• **환경 설정 파일 작성:** n8n의 데이터 저장 위치, 도메인 주소, 인증 키(Encryption Key), 클라우드플레어 터널 시크릿 값 등을 포함한 설정 파일을 작성합니다,. 소스에서는 윈도우를 재설치하더라도 데이터가 날아가지 않도록 **데이터베이스(.sql) 파일과 환경 설정 파일을 C드라이브가 아닌 다른 드라이브(예: D드라이브)에 보관**하는 것을 권장합니다,.
• **서비스 구동 및 상태 확인:** `pm2 start` 명령어를 통해 작성한 설정 파일을 실행하면 n8n과 클라우드플레어가 각각 '온라인' 상태로 활성화됩니다. `pm2 list` 명령어로 현재 실행 중인 프로세스 상태를 확인할 수 있습니다.
• **현재 상태 저장(DUMP):** 가장 중요한 단계로, 서비스가 정상 구동되는 것을 확인한 후 **pm2 save****(또는****pm2 dump****) 명령어를 실행하여 현재 상태를 스냅샷으로 저장**합니다,. 이렇게 해야만 컴퓨터가 재부팅될 때 저장된 정보를 바탕으로 자동 재시작이 이루어집니다.

## 3. 문제 해결 팁

설치 과정에서 환경 변수 문제나 경로 오류가 발생할 수 있습니다. 이때는 에러 화면을 캡처하여 **ChatGPT나 클로드(Claude) 같은 AI 도구에게 질문**하면서 한 단계씩 해결해 나가는 것이 효율적입니다,.

**💡 이해를 돕기 위한 비유** 이 과정은 마치 **'가게 문을 자동으로 열어주는 로봇 점원(PM2)'**을 고용하는 것과 같습니다.
1. **로봇 구입(설치):** 로봇을 사서 가게에 들여놓습니다.
2. **매뉴얼 작성(설정 파일):** "출근하면 전등을 켜고(n8n), 외부 간판을 올려라(클라우드플레어)"라는 상세 지침을 줍니다.
3. **기억 저장(pm2 save):** 로봇에게 이 지침을 절대 잊지 않도록 메모리에 저장시킵니다. 이제 주인인 당신이 밤에 퇴근하고 다음 날 아침에 직접 오지 않아도(컴퓨터 재부팅), 로봇은 저장된 지침에 따라 자동으로 영업 준비를 마쳐놓게 됩니다.
