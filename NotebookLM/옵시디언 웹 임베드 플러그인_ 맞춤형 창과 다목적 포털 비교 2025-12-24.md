# 옵시디언 웹 임베드 플러그인: 맞춤형 창과 다목적 포털 비교

옵시디언에서 웹 애플리케이션이나 웹사이트를 임베드하여 생산성을 높일 수 있는 대표적인 플러그인인 Custom Frames와 Open Gate를 중심으로 비교하여 정리해 드립니다.
1. 주요 플러그인 비교 분석
Custom Frames는 웹 앱을 옵시디언 내의 **맞춤형 창(Pane)**으로 변환하는 데 특화되어 있습니다1. 특히 Google Keep이나 Todoist 같은 인기 서비스에 최적화된 프리셋을 제공하며, 사용자 정의 CSS를 통해 옵시디언의 외관과 조화롭게 스타일을 수정할 수 있는 것이 강점입니다12. 그러나 이 플러그인은 일렉트론(Electron) 환경에서만 작동하기 때문에 모바일 기기에서는 사용할 수 없다는 치명적인 단점이 있습니다3.
반면, Open Gate는 더 넓은 범용성을 제공합니다. 데스크톱에서는 iframe을 사용하고 모바일에서는 webview 기술을 활용하여 모바일 환경에서도 웹 페이지 임베드를 지원합니다45. 또한 사이트의 파비콘(favicon)을 자동으로 감지하여 아이콘을 생성해 주며, iframe으로 임베드가 차단된 일부 사이트도 표시할 수 있는 기능을 갖추고 있습니다56.
Dashboards 플러그인은 앞선 두 플러그인과는 성격이 조금 다르며, 격자(Grid) 구조의 대시보드를 만들어 외부 URL뿐만 아니라 내부 파일이나 코드 블록을 배치하는 레이아웃 구성에 더 집중합니다78.
2. Custom Frames vs. Open Gate 비교표

| 비교 항목 | Custom Frames | Open Gate |
| --- | --- | --- |
| 주요 용도 | 웹 앱을 맞춤형 패널/탭으로 변환12 | 웹사이트를 옵시디언 UI 및 노트에 통합910 |
| 모바일 지원 | 지원 안 함 (일렉트론 기반)3 | 지원함 (webview 활용)45 |
| 배치 위치 | 사이드바(기본), 중앙(설정 시)1112 | 왼쪽, 중앙, 오른쪽 패널 선택 가능513 |
| 주요 프리셋 | Google Keep, Todoist, Notion, Twitter 등14 | Google Calendar, Docs, Trello, YouTube 등15 |
| 특이 기능 | 마크다운 코드 블록 모드 지원116 | 파비콘 자동 감지 및 아이콘 생성56 |
| 커스터마이징 | 사용자 정의 CSS 및 스타일링116 | 사용자 정의 CSS 주입 및 단축키 지원59 |
| 노트 내 임베드 | custom-frames 코드 블록 사용16 | gate 코드 블록 사용517 |

3. 상세 특징 및 차이점
• 설치 및 설정: 두 플러그인 모두 커뮤니티 플러그인에서 설치 후 옵시디언을 재시작해야 설정이 안정적으로 적용되는 경우가 많습니다1819.
• 노트 내 임베드 방식:
    ◦ Custom Frames는 custom-frames 구문을 사용하여 높이(height)나 URL 접미사(urlSuffix)를 설정할 수 있습니다16.
    ◦ Open Gate는 gate 구문을 사용하며, 데이터뷰(Dataview) 명령과 유사하게 URL과 높이를 지정하여 노트 안에 웹 브라우저 창을 직접 삽입합니다1720.
• 사용자 경험: Open Gate는 리본 메뉴에 추가된 아이콘이 사용 후 위치가 이동하거나, 핀(Pin)이 되지 않은 노트 위로 웹 페이지가 덮어씌워지는 등의 사소한 인터페이스 이슈가 보고되기도 했습니다19....
이 두 플러그인은 마치 **옵시디언이라는 성벽 안에 설치된 서로 다른 두 종류의 '문'**과 같습니다. Custom Frames가 특정 귀빈(인기 웹 앱)을 위해 정교하게 장식되고 고정된 **'전용 출입문'**이라면, Open Gate는 언제 어디서든(모바일 포함) 다양한 손님을 맞이할 수 있는 **'다목적 포털'**과 같다고 할 수 있습니다.
