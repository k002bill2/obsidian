# 구글 Conductor: 바이브 코딩을 넘어선 설계 중심 개발

이 동영상은 구글에서 새롭게 선보인 **Conductor**라는 도구를 소개하며, 이를 통해 이른바 '바이브 코딩(Vibe Coding, 별도의 계획 없이 감으로 프롬프트를 입력해 코딩하는 방식)'의 한계를 극복하는 방법을 설명합니다. 초보자분들을 위해 핵심 내용을 다음과 같이 정리해 드립니다.

## 1. Conductor란 무엇인가?

**Conductor**는 구글의 **Gemini CLI**(명령어 라인 인터페이스) 내에서 실행되는 **컨텍스트(맥락) 기반 개발 프레임워크**입니다. 이 도구는 개발자가 코드를 작성하기 전에 요구사항, 제약 조건, 실행 단계를 먼저 구체화하는 '설계 중심 개발'을 돕습니다.

## 2. 왜 '바이브 코딩'보다 좋은가?

단순히 AI에게 한두 번 질문을 던지는 방식은 대화가 길어질수록 AI가 이전 맥락을 잊어버리거나 코드 품질이 떨어지는 문제가 있습니다. Conductor는 다음과 같은 장점이 있습니다:
• **지속적인 기억력:** AI가 알아야 할 프로젝트 정보를 대화창이 아닌 프로젝트 폴더 내의 **마크다운(.md) 파일로 저장**합니다. 덕분에 AI는 프로젝트의 목표와 규칙을 항상 기억할 수 있습니다.
• **기존 프로젝트 분석:** 처음부터 시작하는 프로젝트뿐만 아니라, 이미 만들어진 **기존 코드베이스(Brownfield projects)의 구조와 역사**를 이해하는 데 탁월합니다.
• **팀 단위의 일관성:** 기술 스택이나 워크플로우 기본 설정을 한 번만 정의해 두면, 여러 개발자가 협업하더라도 **마치 한 사람이 짠 것 같은 일관된 코드 품질**을 유지할 수 있습니다.
• **완전 무료:** 구글의 Gemini 모델을 활용하기 때문에 API 비용 걱정 없이 무료로 사용할 수 있다는 점이 큰 장점입니다.

## 3. 주요 작동 방식 (워크플로우)

동영상에서 제시하는 Conductor의 사용 단계는 매우 체계적입니다:
1. **설치 및 설정:** `npm`을 통해 Gemini CLI를 설치하고 Conductor 확장 프로그램을 추가합니다. 이후 `/conductor setup` 명령어로 프로젝트의 기본 토대(기술 스택, 제품 정의 등)를 설정합니다.
2. **설계서(Spec) 작성:** AI가 질문을 던지면 사용자가 답하는 방식으로 프로젝트의 목적과 가이드라인을 작성합니다. 이 과정에서 프로그래밍 언어, 스타일, 프레임워크 등을 매우 세부적으로 지정할 수 있습니다.
3. **트랙(Track) 생성 및 실행:** 새로운 기능 추가나 버그 수정을 위해 '트랙'을 만들고(`new track`), 정교하게 다듬어진 계획에 따라 AI가 자동으로 코드를 구현하도록(`implement`) 명령합니다.

## 4. 핵심 요약 및 효과

Conductor를 사용하면 AI가 단순한 코딩 보조를 넘어 **실행 가능한 사양서와 계획에 따라 움직이는 에이전트** 역할을 하게 됩니다. 이는 코드의 품질과 일관성을 높일 뿐만 아니라, 개발자가 AI에게 일일이 설명하는 시간을 줄여주어 **업무 효율성을 극대화**합니다.

--------------------------------------------------------------------------------

**이해를 돕기 위한 비유:** **'바이브 코딩'**이 요리사에게 "맛있는 저녁 차려줘"라고 막연하게 부탁하는 것이라면, **'Conductor'**를 사용하는 것은 요리사에게 **우리 집 냉장고의 재료 상태, 가족들의 알레르기 정보, 선호하는 요리 스타일이 적힌 상세한 레시피 북**을 건네주고 요리를 시키는 것과 같습니다. 당연히 후자가 훨씬 만족스럽고 일관된 식사를 제공할 것입니다.
