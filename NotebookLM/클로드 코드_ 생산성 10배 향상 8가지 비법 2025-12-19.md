# 클로드 코드: 생산성 10배 향상 8가지 비법

delete
(저장된 대답은 보기 전용입니다.)
Alex Finn의 YouTube 채널에 업로드된 "Master Claude Code in 14 Minutes (8 Steps)" 비디오는 Claude Code를 가장 강력한 AI 코딩 도구라고 소개하며, 사용자가 이 도구를 잘못 사용하고 있을 수 있다고 지적합니다1. 이 비디오는 생산성을 10배 향상시키고 "절대적인 Claude Code 전문가"가 되기 위한 8가지 핵심 비법을 다룹니다1. 이 비법들은 Finn이 직접 연간 30만 달러를 벌어들이는 AI 앱 'Creator Buddy'를 출시하는 데 사용했다고 합니다1.
비디오에서 소개하는 8가지 핵심 비법은 다음과 같습니다:
• 1. 올바른 Claude 규칙 파일(claude.md) 사용:
    ◦ 프로젝트 내에 claude.md 파일을 생성하고 여기에 규칙을 입력하면 Claude Code가 프롬프트를 보낼 때마다 이 파일을 읽습니다1.
    ◦ 이 규칙들은 Claude Code가 모든 작업을 작은 '작업(tasks)'으로 분해하여 생각하고 실행하도록 지시합니다1.
    ◦ Claude Code는 각 작업을 사용자에게 승인받고, 모든 작업 세트가 포함된 폴더를 생성하여 나중에 검토할 수 있도록 합니다1.
    ◦ 작업을 작은 단위로 강제 분해함으로써 Claude Code는 최고의 코드를 생성하며, 이 규칙들을 사용한 후 몇 달 동안 버그가 발생하지 않았다고 합니다1.
• 2. 계획 모드(Plan Mode)의 올바른 사용:
    ◦ 계획 모드는 Claude Code 내에서 가장 강력한 도구로, 원하는 결과를 매번 얻을 수 있도록 보장합니다1.
    ◦ Claude Code를 열고 Shift + Tab을 두 번 누르면 계획 모드로 들어갈 수 있습니다12.
    ◦ Claude Code가 어떤 코드를 작성하기 전에 계획 모드에서 원하는 바를 정확하게 상세히 설명해야 합니다2.
    ◦ 계획 모드를 **"과도하게 사용"**하는 것이 중요하며, 모든 작은 단계(micro step)에 대해 계획 모드를 사용해야 장기적으로 시간을 절약할 수 있습니다2.
    ◦ 계획 모드에서는 **Opus 모델(/mod opus)**을 사용하여 계획을 세우고, 계획 실행 시에는 **Sonnet 모델(/mod sonnet)**을 사용하는 것이 좋습니다2. 이는 계획의 품질을 높이고 비용을 절약하는 데 도움이 됩니다2.
• 3. 체크포인트(Checkpoints)를 위한 GitHub 활용:
    ◦ Claude Code에는 Cursor처럼 특정 체크포인트로 되돌아가는 기능이 없습니다23.
    ◦ 이러한 단점을 보완하기 위해 GitHub를 올바르게 사용하는 것이 핵심입니다3.
    ◦ Claude가 코드를 작성한 후, 코드가 작동하고 효율적이며 좋아 보이면 즉시 GitHub에 커밋해야 합니다3. 하루에 20번까지 커밋할 수 있습니다3.
    ◦ 만약 Claude가 코드를 망치면, 마지막 커밋으로 되돌아간 후 Claude Code로 다시 시작할 수 있습니다3.
    ◦ 새로운 프로그래머라면 GitHub를 워크플로우에 통합하고 즉시 코드를 커밋하는 방법을 익혀야 합니다. 필요하면 Claude Code에게 GitHub 사용법을 물어볼 수도 있습니다3.
• 4. 이미지(Images) 삽입 시점:
    ◦ Claude Code에 이미지를 가져오면 Claude가 이를 보고 코드를 작성할 수 있습니다3.
    ◦ 이미지는 두 가지 경우에 사용하기 좋습니다3:
        ▪ 영감(inspiration)을 제공할 때: 예를 들어, TickTick과 같은 좋아하는 앱의 UI 스크린샷을 가져와 Claude에게 "이 스크린샷에서 UI를 만들어줘"라고 요청할 수 있습니다3.
        ▪ 버그 수정 또는 UI 개선 시: 앱이 이상하게 보이거나 오류가 발생할 경우, 앱이나 오류 화면을 스크린샷으로 찍어 Claude에 넣고 "이 오류를 수정해줘" 또는 "이것을 더 예쁘게 만들어줘"라고 지시할 수 있습니다4.
• 5. /clear 명령 사용:
    ◦ 특히 비싼 요금제를 사용하지 않는 경우 /clear 명령을 자주 사용하는 것이 매우 중요합니다4.
    ◦ AI와 계속 대화하면서 명령을 계속 주면, Claude가 모든 컨텍스트를 받아들이면서 토큰을 많이 사용하게 되어 비용이 급증할 수 있습니다4.
    ◦ /clear는 두 가지 이점을 제공합니다:
        ▪ 환각(hallucinations) 감소: Claude에 보내는 정보의 양을 줄여 환각을 줄입니다4.
        ▪ 비용 절감: 컨텍스트 창의 크기와 전송되는 토큰 수를 줄여 비용을 절약합니다4.
    ◦ Claude가 적당한 크기의 작업을 완료할 때마다 /clear를 사용하여 컨텍스트를 정리하는 것이 좋습니다4.
• 6. 보안 점검(Security Checks) 실행:
    ◦ AI가 생성한 코드는 보안 모범 사례를 따르지 않아 보안상 취약한 코드가 될 수 있으므로, 특히 앱을 인터넷에 배포할 계획이라면 보안 점검이 필수적입니다4.
    ◦ 작업 흐름은 계획 -> 구축 -> 보안 점검이어야 합니다45.
    ◦ Claude가 코드 덩어리나 기능을 완료할 때마다 다음 프롬프트를 사용하여 보안 점검을 실행해야 합니다5:
    ◦ 가장 큰 실수는 API 키나 비밀번호와 같은 민감한 정보를 프런트엔드에 두는 것인데, 이 프롬프트는 모든 정보가 백엔드에 있고 악용될 수 있는 취약점이 없도록 보장합니다5.
• 7. Claude가 구축한 코드 학습:
    ◦ Claude Code를 통해 코딩 경험이 없는 사람도 놀라운 앱을 만들 수 있지만, Claude가 무엇을 구축하는지 이해하는 것이 중요합니다5.
    ◦ 작업 흐름에 "학습(learn)" 단계를 추가합니다: 계획 -> 구축 -> 보안 -> 학습5.
    ◦ Claude가 기능을 완료하고 보안 점검을 마친 후 다음 프롬프트를 사용해 설명을 요청합니다5:
    ◦ 코드를 이해할수록 Claude에게 더 나은, 더 자세한 프롬프트를 줄 수 있게 되며, 이는 더 나은 결과로 이어집니다6.
• 8. Claude Code가 작업하는 동안 시간 현명하게 사용:
    ◦ Claude Code는 놀랍지만, 코드를 구축하는 데 상당한 시간이 소요될 수 있습니다 (15분, 20분, 30분 이상)6.
    ◦ 이 시간 동안 사람들이 쉽게 주의가 산만해져 엉뚱한 행동(예: 둠스크롤링, 유튜브 시청)을 하는 경향이 있습니다6.
    ◦ 해결책은 별도의 AI 챗을 활용하여 아이디어를 생성하고 다른 사업 및 콘텐츠 아이디어를 반영하는 것입니다6.
    ◦ Claude Code가 코드를 작업하는 동안, 이 채팅으로 돌아와 인공지능과 대화하며 현재 Claude Code가 하는 일과 자신의 생각을 공유하고, 아이디어를 교환하고, 다음 단계를 계획하는 데 도움을 받을 수 있습니다6.
    ◦ 이 방법은 생산성을 4,000%까지 높였으며, 정신 건강에도 훨씬 좋다고 합니다7. 규율을 가지고 Claude Code가 작업할 때마다 이 채팅만 사용해야 합니다7.
이러한 8가지 비법과 워크플로우를 따르면 훨씬 더 빠르게 코드를 구축하고 앱의 품질을 10배 높일 수 있다고 비디오는 강조합니다7.
