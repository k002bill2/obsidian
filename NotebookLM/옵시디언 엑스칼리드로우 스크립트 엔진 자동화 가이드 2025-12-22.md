# 옵시디언 엑스칼리드로우 스크립트 엔진 자동화 가이드

Obsidian Excalidraw 1.5.0 버전에 도입된 **스크립트 엔진(Script Engine)**은 사용자가 Excalidraw 내에서 반복적인 작업을 자동화하고 자신만의 매크로를 만들 수 있게 해주는 강력한 기능입니다. 제공된 소스 내용을 바탕으로 상세 내용을 정리해 드립니다.

## 1. 주요 활용 사례 및 자동화 기능

스크립트 엔진을 통해 다음과 같은 작업들을 간편하게 수행할 수 있습니다.
• **요소 연결 및 속성 변경**: 선택한 두 박스를 단축키 하나로 화살표로 연결하거나, 선의 두께(Stroke Width)를 0.5 또는 0.3 등 아주 세밀한 수치로 즉시 변경할 수 있습니다.
• **정밀한 크기 및 위치 조정**: 특정 요소를 원하는 X, Y 좌표에 배치하거나 300x200과 같이 정확한 크기로 설정할 수 있습니다.
• **그리드(Grid) 설정**: 기본 20픽셀인 그리드 크기를 스크립트를 통해 50픽셀 등으로 변경하여 스냅(Snap) 기능을 유연하게 조절할 수 있습니다.
• **요소 그룹화 및 박스 추가**: 선택한 여러 요소(텍스트 포함) 주변에 일정한 여백(Padding)을 가진 박스를 자동으로 생성할 수 있습니다.

## 2. 스크립트 엔진의 특징 및 설정

기존의 Excalidraw Automate 방식보다 훨씬 간편하고 통합된 환경을 제공합니다.
• **스크립트 관리**: 전용 스크립트 폴더를 설정(예: `Excalidraw Scripts`)하면, 해당 폴더 내의 **마크다운(.md) 파일**들을 스크립트로 인식합니다. 단, 이 폴더는 금고(Vault)의 루트 폴더일 수 없습니다.
• **구문 강조(Syntax Highlighting)**: 마크다운 파일 상단에 자바스크립트 주석 처리를 활용한 코드 블록 헤더를 넣음으로써, 옵시디언 내에서 코드 하이라이팅을 지원받으며 쉽게 편집할 수 있습니다.
• **접근성 향상**: 저장된 스크립트는 옵시디언의 **명령어 팔레트(Command Palette)**에 자동으로 등록되며, **단축키(Hotkeys)**를 지정하여 즉시 실행할 수 있습니다.
• **간소화된 코드**: 엔진이 `ea` 객체의 초기화, 리셋, 뷰 설정을 자동으로 관리하므로, 사용자는 핵심 로직인 `run` 부분에만 집중하면 됩니다.

## 3. 스크립트 개발 및 유틸리티

스크립트를 직접 작성하거나 테스트하는 방법은 다음과 같습니다.
• **Input Prompt 활용**: `utils` 객체에서 제공하는 `inputPrompt` 함수를 사용하면, 스크립트 실행 시 사용자로부터 패딩 값이나 선 두께 등의 수치를 입력받는 대화창을 띄울 수 있습니다.
• **개발자 콘솔 테스트**: `Ctrl + Shift + I`를 통해 브라우저 개발자 콘솔을 열고 스크립트를 테스트할 수 있습니다. 이때는 스크립트 엔진과 달리 `ea` 변수 선언, `reset`, `setView` 과정을 수동으로 거쳐야 합니다.
• **작업 흐름**: 뷰에서 요소를 가져오기(`get view selected elements`) → 편집을 위해 복사하기(`copy view elements to ea`) → 속성 수정(예: `strokeWidth` 변경) → 다시 뷰에 추가하기(`add elements to view`) 순서로 진행됩니다.

## 4. 학습 리소스 및 공유

• **공식 문서**: Excalidraw Automate에 대한 상세한 함수 설명, 속성 개요, 스크립트 엔진 설정법 등이 문서화되어 있습니다.
• **스크립트 라이브러리**: 개발자(Zsolt)의 GitHub **'ea-scripts' 폴더**에서 다양한 샘플 스크립트를 내려받거나 내용을 복사하여 자신의 금고에 바로 적용할 수 있습니다.
• **커뮤니티**: 사용자가 만든 스크립트를 GitHub의 Discussions 섹션에 공유하거나 Pull Request를 통해 공식 샘플 라이브러리에 기여할 수도 있습니다.

--------------------------------------------------------------------------------

**비유를 통한 이해**: 이 스크립트 엔진은 마치 **'그림 그리는 로봇에게 줄 수 있는 상세한 매뉴얼'**과 같습니다. 이전에는 로봇의 팔을 일일이 직접 움직여야 했다면, 이제는 "300픽셀 크기의 상자를 그리고 파란색 얇은 선으로 연결해"라는 명령(스크립트)을 적어두고 버튼(단축키)만 누르면 로봇이 즉시 작업을 완수하는 것과 같습니다.
