 ## 📑 PRD: AI싱크클럽 썸네일 및 AI 상세페이지 빌더
1. 프로젝트 개요
프로젝트명: AI싱크클럽 썸네일 및 AI 상세페이지 빌더
목적: 이커머스 셀러(사업가의 무기창고 수강생)들이 상품 정보와 사진만으로 '팔리는 논리'가 적용된 상세페이지와 썸네일을 원클릭으로 제작할 수 있도록 돕는 도구.
핵심 가치: 복잡한 기획 과정을 AI가 자동화하고, 디자인 비용 없이 고퀄리티의 판매 페이지(통이미지)를 생성.
2. 기술 스택 (Tech Stack)
Frontend: React 19 (TypeScript)
Styling: Tailwind CSS (CDN 또는 설치형)
Build Tool: Vite 또는 Create React App
AI Engine: Google Gemini API (@google/genai SDK 사용)
Text Model: gemini-3-pro-preview (기획 및 텍스트 생성)
Image Model: gemini-3-pro-image-preview (이미지 생성 및 Img2Img)
Auth: Google AI Studio API Key (window.aistudio 연동 또는 process.env.API_KEY)
3. 핵심 기능 요구사항
3.1. 공통 기능
API Key 관리: 사용자가 본인의 Google AI Studio 유료 계정 API 키를 연동하거나 시스템 환경변수 키를 사용.
네비게이션: 썸네일 제작 / 상세페이지 제작 탭 전환 기능.
3.2. 상세페이지 기획 및 생성 (Detail Planner) - 핵심
Step 1: 정보 입력 및 설정
기본 정보: 상품명, 카테고리(패션, 식품 등 선택), 가격, 프로모션 정보 입력.
특징 입력: 사용자가 직접 입력하거나 'AI 자동 추천' 버튼을 통해 카테고리/상품명 기반의 추천 특징(USP) 자동 완성.
타겟 설정: 성별(남/녀/전체), 연령대(10대~60대) 다중 선택 UI.
[필수] 레퍼런스 이미지 업로드:
사용자가 보유한 실제 제품 사진을 업로드하는 기능.
업로드된 이미지는 Base64로 변환되어 기획 및 생성 단계의 컨텍스트로 사용됨.
[핵심] 상세페이지 길이(구조) 선택:
Auto (AI 추천): 상품 특성에 맞춰 AI가 길이 판단.
5장 (Short): 저관여/저가 제품용 (핵심 논리 집중).
7장 (Standard): 일반적인 구성 (리뷰/디테일 추가).
9장 (Long): 고관여 제품용 (브랜드 스토리/비교 우위 추가).
Step 2: 전략 기획 (Planning)
AI 기획 에이전트 동작: 입력된 정보를 바탕으로 선택된 길이(5/7/9)에 맞는 논리 구조 설계.
판매 논리 (Seller Winning Logic) 적용:
Hook(후킹) -> Solution(해결) -> Clarity(크기/스펙 비교) -> Social Proof(리뷰) -> Service(활용법) -> Risk Reversal(신뢰/AS).
기획안 UI 출력:
각 이미지 섹션별 Title, Logical Strategy(태그), Key Message(카피), Visual Prompt(그림 묘사)를 카드 형태로 표시.
사용자가 카피(Copy)와 프롬프트를 직접 수정할 수 있어야 함.
언어 제약 사항:
영어 사용 엄격 제한: 생성되는 Key Message는 100% 자연스러운 한국어로만 작성되어야 함 (헤드라인에 Premium, Best 등의 영어 지양).
Step 3: 이미지 일괄 생성 (Generation)
Img2Img 생성:
업로드된 '레퍼런스 이미지'가 있다면 이를 Gemini의 inlineData로 포함하여 원본 제품의 생김새를 유지한 채 배경/연출만 변경.
일괄 처리: 5~9장의 이미지를 순차적 또는 병렬로 생성.
텍스트 렌더링: 생성된 이미지 안에 AI가 기획한 한글 카피(Key Message)가 심미적으로 렌더링 되어야 함.
결과물 뷰어: 세로로 긴 통이미지 형태(9:16 비율)로 미리보기 제공 (스크롤 뷰).
다운로드: 개별 다운로드 및 전체 일괄 다운로드 기능 제공.
3.3. 썸네일 생성기 (Thumbnail Generator)
기능: 정사각형(1:1) 썸네일 생성.
옵션: 스타일(깔끔/라이프스타일/창의적), 요소(손 모델/인물 모델 포함 여부), 텍스트 오버레이(위치 지정).
입력: 상품명, 특징, 원본 이미지 업로드(Img2Img 지원).

5. 프롬프트 엔지니어링 전략 (AI Logic)
개발 시 services/geminiService.ts에 포함되어야 할 핵심 프롬프트 지침입니다.
기획 프롬프트 (Planning):
역할: "한국 스마트스토어/쿠팡 상세페이지 전문 전략가"
논리 분기:
5장: Hook -> Solution -> Clarity -> Service -> Risk Reversal
7장: + Social Proof (리뷰), Detail Deep Dive

# 핵심 기능 요구사항
1. **Gemini API 연동:**
   - `gemini-3-pro-preview` (텍스트 기획용)
   - `gemini-3-pro-image-preview` (이미지 생성용)
   - API Key는 `process.env.API_KEY` 또는 클라이언트 입력 방식을 사용하세요.

2. **상세페이지 기획 모듈 (DetailPlanner.tsx):**
   - **Step 1 (입력):** 
     - 상품명, 카테고리, 특징, 타겟(성별/연령) 입력.
     - **[중요] 이미지 업로드 기능:** 사용자가 제품 사진을 업로드하면, 이후 기획 및 이미지 생성 시 이 사진을 참조(Img2Img)해야 합니다.
     - **[중요] 페이지 길이 선택:** 'AI 추천', '5장(Short)', '7장(Standard)', '9장(Long)' 버튼을 만들고 선택된 길이에 따라 기획 구조를 다르게 가져갑니다.
   - **Step 2 (기획):** 
     - Gemini에게 상품 정보를 주고 5~9개의 섹션 정보를 JSON으로 받아옵니다.
     - 각 섹션은 '후킹-솔루션-입증-디테일-신뢰'의 논리 구조를 따릅니다.
     - **Constraint:** 생성되는 카피(Key Message)는 무조건 '한글'이어야 하며 영어를 최대한 배제합니다.
   - **Step 3 (생성 및 미리보기):**
     - 확정된 기획안과 업로드된 원본 이미지를 Gemini에 전송하여 9:16 비율의 세로 이미지를 생성합니다.
     - 생성된 이미지들은 세로로 이어진 하나의 긴 상세페이지처럼 보여줍니다.

3. **서비스 로직 (geminiService.ts):**
   - `planDetailPage` 함수: 길이(5/7/9)에 따른 논리 분기 처리, 한글 전용 프롬프트 포함.
   - `generateSectionImage` 함수: 텍스트 프롬프트와 Base64 이미지를 함께 전송하여 일관된 제품 이미지 생성.

4. **UI/UX:**
   - 전체적으로 깔끔하고 신뢰감을 주는 블루 톤(Tailwind slate/blue/sky) 사용.
   - 로딩 상태(스피너, 진행률)를 명확히 표시.
   - 반응형 디자인 적용.

위 요구사항을 만족하는 전체 소스 코드를 작성해주세요. (index.html, main.tsx, App.tsx, services/geminiService.ts, components/* 등)
